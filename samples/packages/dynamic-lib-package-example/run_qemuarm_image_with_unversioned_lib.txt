2. The dynamic library should be located in the directory on which it is installed in the Recipe file along with the specified file permissions.
root@qemuarm:~# ls -l /usr/lib/lib_dynamic.so
-rwxr-xr-x    1 root     root          5360 Mar  9  2018 /usr/lib/lib_dynamic.so

3. Since, there are no other Symbolic links generated during the Build, only the real un-versioned dynamic library is present in the "/usr/lib/" directory.
root@qemuarm:~# ls -l /usr/lib/lib_dynamic.so*
-rwxr-xr-x    1 root     root          5360 Mar  9  2018 /usr/lib/lib_dynamic.so

4. However, the Header file is not located in the Image.
root@qemuarm:~# ls -l /usr/include/dynamic_lib.h
ls: /usr/include/dynamic_lib.h: No such file or directory

5. Therefore, even if the library developers do not provide the source code for the dynamic library, the pre-Built dynamic library can be included in the Yocto Build.
	1) The '.so' file can then be directly used on the Image to perform any development on the Image (if required).

6. To exit the QEMU Console, run the below command.
root@qemuarm:~# poweroff
